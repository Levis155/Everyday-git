A Complete Guide To Git

What Is Git?

To truly understand Git, we must first ask ourselves the most obvious question, What is Git?

Git is simply a version control system designed to manage and track changes in source code during software development. It was invented by Linus Torvalds in 2005 for Linux kernel development and it enabled collaboration, history tracking and efficient workflow management

Now, another question arises with this definition. What on earth is a version control system? To answer this in the simplest terms possible: A version control system is like a time machine for our files. It helps us:

  • Track changes to your work over time (like a supercharged "undo" button).
  • Save snapshots of your project at different stages (e.g., "version 1," "version 2," etc.).
  • Collaborate with others without overwriting each other's work.
  • Revert to an older version if you make a mistake.

Some version control system types alongside Git include:

  • Mercurial (Hg)
  • Bazaar (bzr)
  • Fossil

Installing Git

Before installing Git, we must first confirm if it's already installed on our system or not. To do this, head over to the search bar and type in cmd and press Enter to open up Command Prompt.

Once in the terminal, we'll run the command git -v to check if git is installed on our system.

  3. We'll Leave the default configurations as they are clicking on next until we see the install button. We'll click on install, wait for it to finish installing, and walla! Git is finally installed on our system and we can start using it! ðŸ˜€

Configuring Git

Now comes the exciting part! We've installed Git and can now start using Git commands on our terminal.

To start off, we need to tell Git who we are by presenting it with our username and email. To do this, we'll open cmd as demonstrated earlier and run the following two commands:

  • git config --global user.name "Your Name" to present our username.

  • git config --global user.email "project.email@example.com" to present our email.

Next we'll need to confirm that git received our configured username and email using the following commands:

  • git config user.name to check the username

  • git config user.email to check the email

And that's it! Git now knows who we are! ðŸ˜€

Initializing a Git repository

As we learned earlier, Git tracks changes in our files. Therefore, in simple terms a Git repository is just but a folder whose files and subfolders are being constantly tracked and monitored by Git for changes or updates. Git also stores the entire history of these changes allowing us to revert to previous stages of the changes history.

Now you may be wondering How do we get Git to monitor our files? No worries! We'll walk through how to do that in the following steps:

  1. We are going to create a folder on our desktop and open it. Simply right click on your desktop select new, then select folder, and open it.

  2. On the address bar we'll clear the address and type cmd then press Enter to open the directory in cmd.

  3. Once inside cmd we'll run git init command. This transforms New Folder to a Git repository. Congratulations! Git is now tracking your New Folder for changes and updates! ðŸ˜€

    4. Commiting changes

    Commiting chages in Git is like saving a snapshot of your project. So why commit? Commiting a change will help us revert to that version anytime, we'll also pass a message explaining what the changes were as this also helps others see our progress and updates.

    To demonstrate this we're going to add 2 simple text documents to our New folder repository, and for each document, we'll pass a commit message that we added a text document following the steps below:

    1. Open New Folder and inside it create a new text document. Simply right click, select new then Text Document. Name this file text-document1

    2. Now that we've made a change to our New Folder repository (added a text document), we'd like to visualize what change we actually made. To do this open New folder directory in cmd as demonstrated earlier and run the command git status

      Git will tell us that the text document we just created is not tracked, hence we need to add it to the staging area and commit it.

    3. Before commiting the change however, we need to add the change to the staging area. This saves the change. To do this, run the command git add . This command adds all changes to the staging area .

      And now on running git status again we see that text-document1 has been added to the staging area.

    4. Now finally, we can commit the change and pass the message that we added text-document1 using the following command: git commit -m "added text-document1"

      And that's it your first ever commit! ðŸ˜€

    5. For practice, repeat steps 1 to 4 for text-document2.

    6. Now you should have two commits. One for text-document1 and the other for text-document2.

      To view the commits, simply run the command git log --all, and Git shows you all the commits you've made.

    Branches

    The final fundamental concept you'll need to grasp in Git is Branches.

    What is a branch?

    A branch in Git is like a parallel workspace. It lets you experiment, add features, or fix bugs without changing the "main" version of your project. This main version is often referred to as the main or master branch

    In a nutshell:

    • Branch = Copy: Start with the main code, then create your own copy to mess with.
    • Work Safely: Break things here (it won't affect others until you're ready to merge.)
    • Merge Later: When your changes work, glue them back to the main branch.

    Creating, viewing, switching, merging and deleting branches

    To demonstrate how to do all this we'll create 2 new branches on the New Folder repository. We'll then add a text document file to each new branch and call it feature.txt and feature2.txt. Finally, we'll merge one of the new branches to our main branch and delete the other branch in the steps below:

    1. On the New Folder directory in cmd we'll run the command git branch new-branch to create a branch called new-branch

    2. To view all brances present in our repository, we will simply run the command git branch and Git shows us all branches present.

    3. For practice create another branch and view it following steps 1 and 2. Call this branch new-branch2.

    4. Now we'll need to switch to the new branch new-branch and create feature.txt inside this branch by running the command git switch new-branch, and then heading over to New Folder and creating feature.txt

      Finally, add and commit these changes as discussed earlier.

    5. For practice, switch to new-branch2 and create feature2.txt in this branch.

      Remember to add and commit the changes to new-branch2.

    6. To merge new-branch to our main or master branch, we'll first switch back to our master branch by running the command git switch master and then merge new branch to master using the command git merge new-branch

    7. And now finally we'll learn how to delete a branch, by deleting the new-branch2 we created earlier. To do this, simply run the command git branch -D new-branch2
